## Tarefa

Definir o layout base da aplicação de Helpdesk (CRUD de Tickets) com foco em clareza, UX e padrão SaaS moderno.

## Diretrizes de Layout

- Inspirado em dashboards SaaS (Linearm Zendesk, Notion)
- Sem sidebar
- Header simples com título e CTA "Novo Ticket"
- Conteúdo centralizado (max-width ~1100px)
- Listagem em cards
- Mobile-first
- Uso de badges para status e prioridade

## O que NÃO fazer

- Não adicionar animações complexas
- Não usar bibliotecas de UI externas

## output esperado

- Estrutura visual das páginas:
  - Listagem (/)
  - Criação (/novo)
  - Detalhe (/tickets/[id])
  - Edição (/tickets/[id]/editar)
- Hierarquia visual (títulos, ações, conteúdo)
