## Tarefa

Definir a estrutura de pastas do projeto Helpdesk em Next.js (App Router), baseada no layout e UX previamente definidos.

## Contexto

- Aplicação CRUS de Tickets
- Páginas:
  - /
  - /new
  - /tickets/[id]
  - /tickets/[id]/edit
- Uso de Server Components por padrão
- Client Components apenas quando ncessário (forms, filtros, Zustand)

## Requisitos

- Separar coponentes de UI e componentes de domínio
- Separar stores (Zustand)
- Separar validações e tipos
- Estrutura clara e escalável
- Evitar pastas genéricas

## Output esperado

- Estrutura de pastas sugerida
- Breve justificativa das declarações
