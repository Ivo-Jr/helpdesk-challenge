## Tarefa

Implementar uma API mockada para o CRUD de Tickets de Suporte utilizando Router Handlers no Next.js (App Router).

## Contexto

- API usada apenas para simular backend (sem banco real)
- Dados mantidos em memória (array ou Map)
- API será consumida pelo frontend do próprio app

## Escopo da Implementação

### Armazenamento

- Criar um data store em memória (array ou Map)
- Gerar 5-10 tickets inicias de exemplo
- Estrutura do ticket baseada no tipo Ticket já definido no projeto

### Simulações

- Simular latência artificial de ~300ms em todas as requisições
- Simular erro aleatório em ~5% das requisições

## Endpoints Obrigatórios

## GET /api/tickets

- Retornar lista de ticktes
- Suportar filtros via query params:
  - ?status=open
  - ?search=text (buscar por title)
- Retornar status 200

## POST /api/tickets

- Criar novo ticket
- Validar campos obrigatórios de forma simples
- Gerar id e createdAt
- Retornar status 201 com o ticket criado

### GET /api/tickets/[id]

- Retornar ticket pelo id
- Retornar 404 se não existir

### PATCH /api/tickets/[id]

- Atualizar ticket parcialemnte
- Retornar 404 se não existir
- Retornar ticket atualizado

### DELETE /api/tickets/[id]

- Excluir ticket
- Retornar 404 se não existir
- Retornar status 204 (no content)

## Requisitos Técnicos

- Usar Route Handlers (app/api/\*\*/route.ts)
- Usar Request / Response no Next.js

## Output esperado

- Implementação funcional dos endpoints
- API acess;ivel via http://localhost:3000/api/tickets
- Código pronto para consumo pelo frontend
