## Tarefa

Implementar a página de detalhe de um ticket em `/tickets/[id]`.

## Contexto

- Aplicação Next.js (App Router)
- Endpoint disponível: GET /api/tickets/:id
- Página deve ser Server Component
- Estratégia: SSR

## Requisitos

- Criar arquivo app/tickets/[id]/page.tsx
- Buscar ticket por ID no servidor
- Exibir:
  - título
  - descrição
  - email
  - status
  - prioridade
  - categoria
  - data de criação
- Usar badges para status, prioridade e categoria
- Botões:
  - Voltar para listagem
  - Editar (link, sem implementar ainda)
- Tratar ticket inexistente:
  - Exibir mensagem "Ticket não encontrado"
  - Não quebrar a aplicação
- Ao clicar num ticket na pagina de listagem (/), redirecionar para a página do ticket (tickets/[id]) contendo sua descrição.

## O que NÃO fazer

- Não implementar exclusão ainda
- Não criar modal
- Não usar Client Component
- Não adicionar toast

## Output esperado

- Página `/tickets/[id]` funcional
- Renderização server-side
- UX clara para ticket inexistente
